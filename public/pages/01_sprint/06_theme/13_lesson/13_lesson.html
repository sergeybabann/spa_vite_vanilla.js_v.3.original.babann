13. Flex basis, flex-grow и flex-shrink

Вот вам классическая задача с собеседования. Нужно сделать стики футер. Его суть в том, что даже если на странице недостаточно контента, футер не подпрыгивает вверх и не висит посреди страницы, а всегда остаётся внизу.

Не счесть (потому что мы, честно говоря, и не считали) количество начинающих верстальщиков, которые заваливают эту задачу на собеседовании. Но вы в их число уже точно не попадёте. Потому что сейчас узнаете простой способ сделать так, чтобы футер всегда был внизу страницы.

На задачу можно посмотреть иначе. Незачем прилеплять футер к низу. Нам просто нужно, чтобы контент занял всё свободное пространство. В итоге футер окажется внизу — ему просто некуда больше деться.

Значит, для решения задачи нужно перераспределить свободное пространство на странице. Сделать это можно с помощью свойства flex-grow. Но для его работы нужно подготовить условия:

  1. Сделайте body флекс-контейнером.
  2. Измените направление внутри контейнера на column.
  3. Укажите минимальную высоту body — 100%.
  4. Добавьте блоку .main свойство flex-grow и установите значение 1.

Когда вы задали блоку .main свойство flex-grow: 1, то отдали ему всё оставшееся свободное пространство по высоте. В итоге футер оказался прижатым к низу страницы.

Свойство flex-grow указывает, можно ли флекс-элементу расшириться при наличии свободного места и насколько. Его значением может быть любое положительное целое число. По умолчанию значение равно 0 — расширяться нельзя.

Если всем флекс-элементам задать flex-grow: 1, то свободное пространство равномерно распределится между ними. Но стоит нам задать одному элементу flex-grow: 2, как он заберёт себе в два раза больше свободного пространства, чем элементы с
flex-grow: 1.

Нет, это только начало. Дальше — ещё интереснее. Возьмём ещё одну распространённую задачу.

На странице размещены карточки с эпизодами подкаста. Пока вышло пять выпусков. Нужно сделать так, чтобы в первом ряду разместились три эпизода, а две оставшиеся карточки заняли всю ширину родительского контейнера во втором ряду.

Решить эту задачу поможет свойство flex-basis. Оно указывает размер элемента до того, как на него начнёт действовать расширение.

Значением flex-basis может быть размер в любых относительных или абсолютных единицах. Значение по умолчанию — auto. В таком случае элемент занимает столько пространства, сколько требуется для отображения его контента.

Добавьте элементу .card базовый размер 300px. Затем разрешите карточкам занимать свободное пространство, если оно есть.

И это ещё не всё. Можно управлять не только расширением, но и сжатием элементов. Для этого используется свойство
flex-shrink. Это полная противоположность свойству flex-grow.

Значением flex-shrink может быть любое положительное целое число. Отрицательные значения игнорируются. По умолчанию его значение 1 — то есть все элементы сжимаются с одинаковой скоростью. Чем больше значение flex-shrink, тем быстрее сжимается элемент.

Положительные значения рассмотрели, но не разобрались с нулём. Элементы, у которых установлено свойство flex-shrink: 0, не сжимаются. Их размер не может стать меньше, чем тот, что задан с помощью flex-basis.

Чтобы убедиться в этом, установите второй карточке flex-shrink: 0.

Значение по умолчанию — flex: 0 1 auto. Расшифровывается это так:
